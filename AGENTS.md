# AGENTS.md

# コミュニケーション
- 日本語で応答する（コード・変数名は英語）
- 簡潔に回答し、自明な説明は省略する
- 複雑なタスクでは実装前に計画を提示し、承認後に着手する

# フォルダ構成
- **docs**: ドキュメントはすべてこのフォルダ内に配置する
- **backend**: バックエンド部分の実装はこのフォルダ内に配置する
- **frontend**: フロントエンド部分の実装はこのフォルダ内に配置する

# コーディング規約
- **インデント**: スペース2つを使用する（タブ不可）
- **命名規則**: 変数・関数名はキャメルケース（例: `getUserName`）、定数はアッパースネークケース（例: `MAX_RETRY_COUNT`）
- **関数の長さ**: 1つの関数は50行以内に収める。それを超える場合は分割を検討する
- エラーは握りつぶさず、意味のあるメッセージ付きで処理する
- 共通化できる部分は共通化する(可読性重視)
- スタイルは画面ごとのCSSファイルで管理
- テストファイルは必ず作成する

# Git規約
- Conventional Commits形式、本文は日本語（例: `feat: ユーザー認証にOAuth2を追加`）
- 確認なしに自動コミット・自動pushしない

# 禁止事項
- README・ドキュメントを勝手に変更しない(作成は可)
- テストコードを確認なしに削除・コメントアウトしない
- 既存の動作するコードを理由なくリファクタリングしない
