# テスト仕様書

## 1. 概要

### 1.1 対象アプリケーション

タスク管理アプリケーション

### 1.2 技術スタック

| レイヤー | 技術 |
|---------|------|
| バックエンド | Express.js, TypeScript, better-sqlite3 |
| フロントエンド | React 18, TypeScript, Vite, Tailwind CSS, React Router |
| テストフレームワーク | Vitest, supertest (Backend), @testing-library/react (Frontend) |

### 1.3 テスト種別

| テスト種別 | 対象 | テスト数 | テストファイル |
|-----------|------|---------|--------------|
| バックエンド単体テスト | API エンドポイント | 47 | `backend/src/routes/tasks.test.ts`, `backend/src/routes/comments.test.ts`, `backend/src/routes/projects.test.ts` |
| フロントエンド単体テスト | React コンポーネント | 27 | `frontend/src/pages/TaskList.test.tsx`, `TaskCreate.test.tsx`, `TaskDetail.test.tsx`, `TaskEdit.test.tsx`, `frontend/src/pages/ProjectList.test.tsx` |
| E2E テスト（手動） | ブラウザ操作による全画面検証 | 11 | `test-plan.md`（テスト計画書） |

### 1.4 テスト実行方法

```bash
# バックエンドテスト
cd backend && npx vitest run

# フロントエンドテスト
cd frontend && npx vitest run

# E2Eテスト（手動）
npm run dev  # アプリ起動後、ブラウザで http://localhost:5173/tasks にアクセス
```

| タスク一覧（GET /api/tasks） | project_id (query) | 任意、正の整数 | "プロジェクトIDは正の整数で指定してください" |

| プロジェクト更新（PUT /api/projects/:id） | name | 必須、255文字以内 | "プロジェクト名は必須です" / "プロジェクト名は255文字以内で入力してください" |
| プロジェクト系エンドポイント | id | 正の整数 | "プロジェクトIDは正の整数で指定してください" |
